****Campus Network Segmentation and Inter-VLAN Routing

Project Description****

This project demonstrates the design and implementation of a small enterprise/campus network using Cisco Packet Tracer. The network is segmented into multiple subnets (192.168.10.0, 192.168.20.0, and 192.168.30.0) connected through a central router, enabling communication between different departments while maintaining logical network separation. The topology includes routers, switches, PCs, and VLAN configuration to simulate a real-world organizational network.

**Features**

1.Multi-subnet network architecture

2.Router-based inter-network communication

3.VLAN implementation for network segmentation

4.Multiple switches and end devices

5.IP addressing and subnetting

6.Scalable enterprise network design

7.End-to-end connectivity testing using ping and simulation mode

**Technologies Used**

1.Cisco Packet Tracer

2.Cisco Router (Router-PT)

3.Cisco Catalyst 2960 Switches

4.VLAN Configuration

5.IPv4 Addressing

6.Routing and Switching Concepts

7.Network Simulation Tools


**Network Structure**

Network	Purpose
192.168.10.0/24	Department A LAN

192.168.20.0/24	Department B LAN

192.168.30.0/24	Department C/VLAN LAN


**How to Run**

1.Install Cisco Packet Tracer.

2.Clone this repository:

git clone https://github.com/BADGUY2004/campus-network-segmentation.git

3.Open the .pkt file in Cisco Packet Tracer.

4.Switch to Realtime mode.

5.Verify device configurations (IP addresses, VLANs, and router interfaces).

6.Test connectivity using:

ping 192.168.20.2

7.Use Simulation Mode to observe packet flow across the network.
